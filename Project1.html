Project 1: Creating a website

In the first project the requirements are to choose or create a template for our website. It has to be well organized, contain information about ourselves and our resume.
This website also acts as a logbook for this project and the ones to come. It is hopefully detailed so successors can replicate the steps that were taken accurately. (N.b. this website is created in the spring of 2023, some of the software used here might have become outdated by the time you read this.)

The making of this website

There were a few steps in the construction of this website. The steps can be categorized into one of the three following categories:

Searching for data
Downloading
Installations and setup

Creation of a Github account

When signing up to Github you are required to give your mail, create a password and choose a suitable name for your account. If you have done that you will get a verification email and it is time to choose what type of license you are going to get for yourself. I got a student license, it is free, given that you can provide proof of studying this semester at a university. There is also a professional license but it is not free, and it is mainly used in large corporations.
This should be a fairly straight forward process to most people nowadays.

Making a repository

When you have logged into your github account on the github page you will be met with some suggestions and instrucctions on what to do. When you have checked that out (or skipped it), find a green button in the top left that looks a bit like this one:

New

When you have pressed that button you will be met with a screen that looks like the photo here below. I followed this tutorial by Tony Teaches Tech and he suggested that we put the name of our github account, in my case Tarnarsson and then .github.io for our first repository so we get Tarnarsson.github.io and put it in the red box. For this simple website I had to have it public as can be seen underlined. Then we can keep the default settings, if you want you can create a README file, that is a summary of the page but there is no need for it if you are the only one that is working on the code it self. When you are ready you can create the repository.

When you have created the repository you will find this on you screen and we are going to use the link in the red box 'uploading an exsisting file', but before you do that you have to chose a template and set it up on your computer.

Setting up html template

First I went to this html5 website that has a few html templates for free. I chose this template called Zerofour because it had the characteristics that I wanted for my website. Next I downloaded the file and found it a suitable working directory, I am working on a macbook and so I renamed my file Tarnarsson.github.io, the same as my website then stored in this directory "User/tarnarsson/Github/Tarnarsson.github.io"

Finishing touches and website updates

I do al of the changes to my websites through isual Visual Studio code (VS code), the reason being that it was already on my computer and I am familiar with it. It lets me edit my code and keep multiple different types of files open at the same time, provides suggestions and has multiple other handy properties. I change my files locally first and then push them to the cloud, as I am the only one working on the website I do not have to use any pull commands to get the newest versions from the cloud. When I have changed any of my html files i run the following commands:

git add .
git commit -m 'Message that describes the changes made'
git push .

Unfortunately there seems to be some kind of misalignment between my vs code and the website so the push command does not push the code to the repository. So every time I go to the Github Desktop app and press this button that updates the website.

Update:

This problem was easily fixed. In stead of doing "git push ." I simply wrote "git push". The bash requested the log in for the github account and then the password. After that I just use this to update the website.

git add .
git commit -m 'Message that describes the changes made'
git push
